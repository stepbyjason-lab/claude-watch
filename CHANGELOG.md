# Changelog

All notable changes to `claude-watch` are documented here.

## [Unreleased] — fork `stepbyjason-lab/claude-watch`

Additive changes on top of upstream `devinilabs/claude-watch`. Classic extraction and existing library caches are unchanged.

### Added
- **Time-aware merge is now value-mismatch gated — a frame only folds away if the previous frame's content still LOOKS THE SAME (pixel value, not just present) in it.** Cross-deck dogfood on a sparse, uniform-template deck (dark background, one centered title line per slide) found the previous merge rule (`gap < --merge-gap` and `dist < --merge-dist`) alone was not enough: two *distinct* slides on that template differ only in the title strip, so their whole-frame dhash distance lands at 5-10 — squarely inside the default merge band. All 10 of 10 merges on that clip were false, silently destroying ~29% of the deck's slides (see [`docs/dogfood/2026-07-02-merge-defaults-cross-deck.md`](docs/dogfood/2026-07-02-merge-defaults-cross-deck.md)), and no dhash distance threshold can separate the two cases — the same dist-5 gap was a legitimate build-step on a different, content-dense deck. A first attempt at a `vanish_ratio` check (mask containment: does *any* content exist near the anchor's content) still failed in the field — same-position titles on the uniform-template deck made two distinct slides' content masks overlap almost completely, scoring a real false-merge pair as low as 0.005. The check was retuned on 15 verified pairs across three real decks to compare pixel *values*, not just mask presence: for every content pixel in the anchor, is there a close-value match anywhere in a 3x3 neighborhood of the same position in the candidate (1px shift/antialiasing tolerance)? A build/scroll step is additive (content stays byte-similar, more is added) and scores near 0; a slide change — including same-position replacement — leaves no matching value nearby and scores near 1. A pair that passes the existing gap+dist band now also needs `vanish <= 0.08` (`MERGE_MAX_VANISH`, field-tuned on three decks: the false-merge class floor measured 0.153, the lowest legitimate build-step measured 0.058, and 0.08 sits between them, recall-biased) to actually fold; if the vanish check itself fails (ffmpeg/subprocess error), the pair fails safe — warned and rejected (kept), never silently merged on missing evidence. Rejected pairs aren't dropped from the record: dist 5-10 is already inside `phash_dedup`'s near-dup flag band (default `drop_dist=4 < d <= flag_dist=10`), so a rejected merge still surfaces as a `review: near-dup` line — recall-first. **Effect:** sparse uniform-template decks stop losing slides to false merges, including the same-position-title case that defeated the first (mask-only) version of this gate; genuine scroll-steps that move existing content off-frame (rather than adding to it) now also stay as kept, near-dup-flagged frames instead of silently folding — more frames to read on that narrower class of build-step, in exchange for zero silently-lost slides. **Unchanged:** `--merge-gap 0` (or `--merge-dist 0`) still disables the entire merge pass, byte-identical to before; the `dist == --merge-dist` exact-threshold preserve-and-flag path (R08) is untouched and does not consult the new gate. No new CLI flags.
- **Audit lines (`review: near-dup`, `review: merge-threshold`, `merged:`) now reference only surviving frames.** These lists are built inside `detect_slides_freeze` before the frame set is finalized, so a frame they reference can vanish afterward: the time-aware merge can fold away one member of a `phash_dedup`-flagged near-dup pair (a near-dup is hash-close, so it overlaps the merge pass's candidates), and `--prefer-light` can drop a frame post-merge. Left unfiltered, a printed review line can point at a frame that's no longer on disk or in the manifest — observed on the R08 forum clip, where a `review: near-dup` line named a frame that had just been folded away by `merged:`. Now `detect_slides_freeze` re-validates all three lists against the final surviving frame set right before returning: `flagged`/`merge_flagged` lines (which ask the reader to open **both** frames) are dropped unless both timestamps survive; `merged:` lines (a drop notice whose second timestamp is *always* absent by design) are dropped only if their anchor timestamp doesn't survive. Net effect on the default path (merge on, `--prefer-light` off): stale `review: near-dup` lines that were about to be folded away by the merge pass are now suppressed — fewer near-dup lines in stdout than before, with no change to which frames are extracted. Unchanged: frame extraction/recall/count and manifest contents — only the printed audit lines are filtered; `--merge-gap 0` (or `--merge-dist 0`) with `--prefer-light` off stays byte-identical to R05 (no frame is ever unlinked after dedup, so the filter is a no-op).
- **Time-aware merge now preserves + flags exact-threshold pairs instead of folding them.** A pair landing *exactly* on `--merge-dist` was the most fragile call the merge pass made — one hamming-distance unit separates "same build step" from "different slide", and a small crop change is enough to flip it. Field evidence: on a real Zoom clip, a fold at dist-exactly-11 dropped a genuine prepared slide (deck page 16), while dist-9 folds on the same clip were safe; a ~6px difference between two independently-measured, both-correct crops of the same recording was enough to move that pair from dist 12 (kept) to dist 11 (folded) — see [`docs/dogfood/2026-06-28-auto-crop-field-limits.md`](docs/dogfood/2026-06-28-auto-crop-field-limits.md), "2026-07-02 follow-up". Now `dist == --merge-dist` (with `gap < --merge-gap`) keeps the frame instead of merging it, and prints a `review: merge-threshold t=A ~ t=B (dist D, gap Gs)` line so the notes step can judge the pair directly (fold it as a ledger entry if it really is the same build step, keep both as real content otherwise). The preserved frame becomes the new merge anchor, same as any other kept frame. `dist < --merge-dist` still folds exactly as before, and `--merge-gap 0`/`--merge-dist 0` still disable the pass entirely (byte-identical to the pre-merge R05 path). Net effect: frame counts can rise on decks with threshold-exact pairs (the forum clip from the crop-limits doc went 19 → 21 with no other flag changes).
- **`--probe-frame`/`--probe-at` — automate the "get one frame" step of LLM-assisted crop measurement.** Previously, measuring a `--slides --crop W:H:X:Y` rectangle by eye meant running the full `--slides` pipeline just to get one frame — paying for a whole detect/extract/transcribe pass while the frames it produced were already cropped wrong (default `--cam-corner`). `--probe-frame` (slides only) downloads the source, extracts a single native-resolution frame via the existing `extract_frames`/`probe_dimensions` helpers, and prints a structured `=== probe frame ===` block (frame path, timestamp, `source_resolution: WxH`, and the re-run command) then exits — no detect, no extract, no transcribe. `--probe-at TIMESTAMP` picks the capture point (`SS`/`MM:SS`/`HH:MM:SS`); default is 25% into the video (layout chrome — cam/chat/toolbar position — is fixed for the whole call, so an exact timestamp doesn't matter as long as it avoids the intro/waiting-room start and outro/Q&A end). Validation fails fast: `--probe-at` without `--probe-frame`, `--probe-frame` without `--slides`, or `--probe-frame` combined with `--start`/`--end` all exit with a clear message. `meta.json` is written (so a re-probe with the same flags reuses the cached download) but `manifest.json` is not (the pipeline never ran to completion). The crop rectangle itself is still measured by the caller (Claude or a subagent) via pixel-boundary sampling on the returned frame — this only automates frame acquisition, not measurement.
- **`--merge-gap`/`--merge-dist` — time-aware merge of animation build-steps (slides freeze).** A single `--hold` can't fit a deck that mixes very short (3–6s) and very long (~2min) slides: raise it and short slides are missed, lower it and each animation build-step (a bullet appearing, a panel scrolling) is logged as a separate slide. After dedup, this pass folds a frame into the previous kept one iff it is BOTH within `--merge-gap` seconds (default 15) AND within `--merge-dist` hash-distance (default 11) — the same screen evolving, not a new slide. A plain `--phash-dist` can't express this (it's time-blind). So you lower `--hold` for recall and let the merge collapse the build-steps. Merges are reported as `merged: t=A ~ t=B (dist D, gap Gs)` lines (nothing vanishes untracked), and the pass reuses the dedup hashes (no double ffmpeg). Set either knob to 0 to disable (byte-identical to the pre-merge path). Freeze-only; ignored under `--detect scene`. Field-validated on a 12-min Zoom clip: `--hold 3` went 22 → 19 (3 build-pairs merged; the 4 short slides `--hold 6` missed were preserved). Defaults are tuned on one deck — watch the `merged:` lines and tune per presentation style.
- **`--crop auto` — automatic slide-region detection for slides freeze mode.** Instead of measuring `--crop W:H:X:Y` by hand, `--crop auto` samples grayscale frames via ffmpeg, builds a per-pixel temporal motion map (summed frame-to-frame abs-diff), and trims persistently high-motion edge bands (presenter cam / chat / toolbar) to localize the static slide region, even-aligned and re-validated through `parse_crop` before use. Zero-dependency (ffmpeg + a stdlib PGM parser — no numpy). Best-effort and fail-open: when the source is full-screen, demo-heavy, or too uniform to localize a stable region (trimmed area <40% or >92% of the frame), it returns no crop — warning and falling back to `--cam-corner`/`--caption` — so explicit `--crop W:H:X:Y` stays the precise option. ffmpeg/PGM failures warn before falling back (no silent give-up). Field-tested on two real 2560×1600 recordings (a demo-heavy seminar and a Zoom gallery forum with continuously-changing slides beside talking-head cams); both correctly fell back rather than emit a wrong crop. It works best when the moving chrome forms a clear edge band beside a comparatively static slide — talking-head Zoom layouts, where a moving face reads as slide-like content, still need manual `--crop`. Signals tried and the limit: [`docs/dogfood/2026-06-28-auto-crop-field-limits.md`](docs/dogfood/2026-06-28-auto-crop-field-limits.md).
- **`--prefer-light` brightness filter for slides freeze mode (opt-in).** After freeze capture + dedup, drops frames whose mean grayscale is below `--light-threshold` (0–255, default 80) — a cheap heuristic that discards dark IDE/terminal *demo* screens, cutting downstream LLM token cost. Assumes light-background slides (off by default; leave off for dark-themed decks, whose slides would be dropped). Brightness measurement that fails on a frame keeps it (fail-open with a warning); if every slide is dropped, a warning is emitted (no silent-empty). It removes dark demos only — light-background demos still pass, so final slide selection remains the notes step's job. Validated on a 155-min Zoom seminar: a pure live-demo stretch (mean luma 2–74) went from 8 frames to 0 (with a "dropped all" warning), while slide stretches were unaffected.
- **`--slides` freeze-based detection (new default `--detect freeze`).** Slides mode now captures one frame per *held* (static ≥ `--hold`s, default 5s) screen via `ffmpeg freezedetect`, instead of scene-cut + coverage-floor. A prepared slide holds; a live demo (scrolling code/browser) never settles — so demo scroll-noise is skipped and the output count tracks held screens, not video length (no candidate-cap blowups on multi-hour recordings). Output frames are cropped to the slide region (fewer pixels → lower downstream LLM token cost). New flags: `--detect freeze|scene`, `--crop W:H:X:Y` (explicit slide region — needed for Zoom-style recordings where the cam/chat/taskbar isn't in a corner; freeze needs the moving chrome out of frame), `--hold SECONDS`, `--freeze-noise <dB|0..1>` (dB must be negative), `--candidate-cap N` (the previously-hardcoded 800 cap, now exposed and applied in both modes). The legacy scene-cut + coverage-floor detector is preserved byte-identical via `--detect scene`. Freeze removes scroll-noise but does not classify slide-vs-held-demo — that content judgment is left to the notes step. Crop/freeze-noise inputs are validated before reaching the ffmpeg filtergraph; all failure surfaces (unrunnable binary, no/empty SRT, bad knobs, ffmpeg error, partial-extraction cleanup) degrade or signal cleanly. Measured on a 155-min Zoom seminar: scene mode = 465 frames (mostly scroll-noise), freeze = 171 distinct held screens.
- **Local Whisper backend (preferred over the cloud).** When no native captions exist, claude-watch now uses a local Whisper before falling back to Groq/OpenAI. A `whisper` (openai-whisper) or `whisper-ctranslate2` (faster-whisper) on PATH is auto-detected (language auto-detected; `WHISPER_MODEL` picks the model, default `base`); an isolated venv or custom wrapper is wired via `WHISPER_LOCAL_CMD` (read from the OS environment or `~/.config/claude-watch/.env`, with `{audio}`/`{outdir}` placeholders — whichever is absent is appended). Resolution order: forced `--whisper local|groq|openai` > `WHISPER_LOCAL_CMD` > PATH CLI > Groq > OpenAI. If an auto-picked local backend fails, it falls back to the cloud (a forced backend does not). The command's `.srt` output is parsed by the existing VTT grammar. Failure surfaces (unrunnable binary, nonzero exit, no/empty/unreadable SRT, unparseable `WHISPER_LOCAL_CMD`, ffmpeg audio-extraction error) all degrade gracefully to a cloud fallback or frames-only notes instead of crashing; subprocess output is control-char-sanitized before it reaches stderr. `setup.py --check` reports `whisper_backend: local` and treats a local Whisper as satisfying the transcription requirement (no cloud key needed).
- **`--slides` mode** — high-recall capture of a lecture deck (aims for every prepared slide; fast-flips and visually-similar slides can still be missed — see [`docs/dogfood/2026-06-05-detailpage-slides.md`](docs/dogfood/2026-06-05-detailpage-slides.md)): crops out the presenter cam + caption, scene-detects on the slide region at a low threshold, and conservatively deduplicates near-identical frames via a zero-dependency `ffmpeg` 9×8 **difference (edge) hash** — chosen over an average hash because it tells apart monochrome text slides on white decks, where an average hash collapses them (borderline pairs are kept and flagged, not silently dropped). Native 720p extraction. Flags: `--slides`, `--cam-corner`, `--caption`, `--hi-res`, `--phash-dist`.
- **Slides end-of-video tail anchor** — `apply_coverage_floor(..., include_tail_anchor=True)` (slides mode opts in; classic mode keeps the upstream default, byte-identical) adds one extractable floor candidate near `duration − 0.5s`, so the final slide — shown past the last coverage step, too late to register as a scene cut — gets coverage. Validated on a 38-min white deck (last slide recovered, 25 → 26); for dense text decks `--scene-threshold 0.15 --phash-dist 2` lifts capture further (25 → 31) without ballooning. Evidence: [`docs/dogfood/2026-06-05-harness-dhash-validation.md`](docs/dogfood/2026-06-05-harness-dhash-validation.md).
- Slides cache identity: the full detection profile is folded into the slug (any slide flag change re-runs cleanly); default-mode slugs hash identically to upstream.
- Hardening: `-protocol_whitelist file` on all ffmpeg inputs, `urlparse`-based source-scheme allowlist, candidate-frame cap, UTF-8 stdout/stderr (no cp949 crash on Windows), frames-wipe containment guard (never deletes outside the resolved library root), and local-source validation (`copy_local` refuses non-files and non-video extensions before creating the library symlink).

### Changed
- **Library root relocated to the OS app-data dir.** The hardcoded `~/claude-watch/library`
  (a folder dumped into the home-directory root on every platform) is replaced by a resolver:
  Windows `%LOCALAPPDATA%\claude-watch\library`, macOS `~/Library/Application Support/claude-watch/library`,
  Linux `$XDG_DATA_HOME` (or `~/.local/share`) `/claude-watch/library`. Override order:
  `--out-dir` flag > `CLAUDE_WATCH_LIBRARY` env var > the same key in `~/.config/claude-watch/.env` >
  legacy `~/claude-watch/library` if it already exists (existing installs keep working untouched) >
  platform default. `setup.py` scaffolds the new `.env` key as a commented hint.
- **Notes are now concept-first.** The scene-first "one scene/slide = one section" template (On screen + Said + Synthesis per scene) was replaced by a concept-first study-notes contract: core thesis, concept map, learning path, frameworks/examples/caveats, inline visual evidence with captions, and a slide coverage ledger — gated by a pre-write quality check. Screenshots and timestamps are evidence, not the main structure. Script/extraction output is unchanged.
- **Slide coverage is now auditable and embed-first.** Because `--slides` (freeze) extraction is high-recall, its output mixes prepared slides, held demo/app screens, and the occasional non-content still. The notes contract now classifies *every* extracted frame in the Slide Coverage Ledger as `inline` / `ledger` / `non-slide` (held demos and non-content stills carry a one-line reason), with the audit identity `extracted = inline + ledger + non-slide` so nothing is silently dropped. Statuses are dispositions, not frame kinds: prepared slides default to `inline` (ledger only for genuine near-duplicates), and `non-slide` is a narrow exception, not a dumping ground — a long deck can no longer quietly collapse to a handful of embeds. Ambiguous taught-from screens (a doc/browser the speaker teaches from) are treated as slides. Doc/contract change only; script/extraction output is unchanged.

## [0.1.0] — 2026-05-03

### Added
- `/claude-watch <url-or-path> [topic]` slash command that produces structured study notes.
- Scene-aware frame extraction: ffmpeg scene detection (default threshold 0.30) with a coverage floor (synthetic boundaries every 45s across long static gaps) and a budget cap (default 80 frames, drops lowest-scoring detected scenes first; floor boundaries are always preserved).
- Persistent library at `~/claude-watch/library/<slug>/` with cached download, transcript, and scenes — re-runs only regenerate frames + notes.
- Slug rule `YYYY-MM-DD-<title>-<sha1(source+focus)[:4]>` so chronological + collision-safe across focus-range re-watches.
- Native caption pull via yt-dlp (manual + auto-subs) with VTT dedupe.
- Whisper fallback: Groq `whisper-large-v3` (preferred), OpenAI `whisper-1` (alt). Stdlib HTTP clients — no SDKs.
- `--start`/`--end` focused mode with denser coverage floor (15s vs 45s default).
- `setup.py` preflight (`--check` / `--json`) with cross-platform installer (`brew` on macOS auto-runs; `apt`/`dnf`/`winget`/`pip` commands printed elsewhere).
- Three-surface distribution: Claude Code plugin, claude.ai `.skill` bundle (built by `scripts/build-skill.sh`), Codex skill.
- SessionStart hook prints a one-liner only when remediation is needed.
- Strict notes template baked into SKILL.md: TLDR, Key Concepts, per-scene Notes (On screen + Said + Synthesis), Code & Commands, Diagrams Referenced, Open Questions.
